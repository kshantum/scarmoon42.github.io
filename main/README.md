# Система управления кадровой политикой СибГИУ

Комплексная информационная система для управления кадровой политикой Сибирского государственного индустриального университета (СибГИУ), включающая оценку преподавателей, проведение открытых занятий и управление учебными материалами.

## 🏗️ Архитектура и Технологический стек (Монолитная архитектура)

Проект строго следует современному технологическому стеку:

### Бэкенд
- **Среда выполнения:** Node.js 18+ (LTS)
- **Язык:** TypeScript 5+
- **Основной фреймворк:** React Router v7 (framework режим) / Express
- **ORM:** Prisma 5+
- **Дополнительно:** `zod` (валидация), `helmet` (безопасность заголовков)

### Фронтенд
- **UI-фреймворк:** React 18+
- **Стилизация:** Tailwind CSS 3+
- **Библиотека компонентов:** `@shadcn/ui`
- **Тестирование:** Jest + React Testing Library

### Базы данных
- **Основная СУБД:** PostgreSQL 17+
- **Кэширующая СУБД:** Redis 7+

### Инфраструктура
- **Контейнеризация:** Docker & Docker Compose
- **Сетевые взаимодействия:** REST / GraphQL
- **Мониторинг:** Prometheus + Grafana
- **Логирование:** Loki
- **Аутентификация:** Keycloak

## 🚀 Быстрый старт

### 1. Единый запуск всего стека через Docker
Теперь проект поставляется **полностью готовым к контейнеризации**, и само приложение (Бэкенд + Фронтенд как монолит) поднимается вместе с базами.

```bash
docker-compose up -d --build
```
Это запустит:
- **Приложение (App)** — доступно на `http://localhost:3001` (или на `http://localhost:3000` благодаря переадресации).
- СУБД: PostgreSQL (порты 5432) и Redis (6379).
- Авторизацию: Keycloak (8080) и его БД (5433).
- Инфраструктуру: Prometheus (9090), Grafana (3002) и Loki (3100).

Ожидайте ~30-60 секунд для инициализации баз данных.

## 🔐 Аутентификация и Keycloak

Проект использует Keycloak для безопасной аутентификации на основе JWT (Bearer токенов) и разделения ролей.

1. Откройте **http://localhost:8080** (Admin Console: `admin` / `admin`).
2. Создайте Realm с названием **`app`** (см. `KEYCLOAK_REALM` в `.env`).
3. В разделе Clients создайте клиента **`frontend`** (Client authentication: On). В "Valid redirect URIs" добавьте `http://localhost:3000/*` и `http://localhost:8080/*`. В "Web origins": `http://localhost:3000`, `http://localhost:3001`.
4. Скопируйте `Client secret` со вкладки Credentials клиента и обновите `KEYCLOAK_CLIENT_SECRET` в `.env.development`.
5. Создайте роли: `admin`, `teacher`, `student`, `secretary`, `expert`. Создайте тестового пользователя и назначьте ему роли.

## 👥 Роли пользователей и Навигация

В системе реализованы строгие потоки (flows) для разных ролей:

1. **Преподаватель (teacher)**: Прямая навигация к выбору должности → Загрузка УМК, КП и документов о повышении квалификации (ПК) → Тестирование → Доступ к QR-коду открытого занятия.
2. **Эксперт (expert)**: Выбор кандидата для оценки → Просмотр документов → Заполнение чек-листов и оценочной анкеты (в т.ч. через GIFT формы) → Оценка открытого занятия.
3. **Секретарь (secretary)**: Управление пользователями, назначение открытых занятий, просмотр итогового рейтинга. Поддерживается создание *временных аккаунтов*.
4. **Студент (анонимно)**: Доступ по QR-коду к чек-листу для оценки текущего открытого занятия (от 0 до 3 баллов по 5 критериям).

> *Примечание: Выход из аккаунта доступен на **всех** страницах для всех авторизованных ролей с полным сбросом состояния.*

## 📂 Работа с файлами и форматом GIFT

Система поддерживает полнофункциональную загрузку с изоляцией файлов по пользователям и проверкой типов:

### Учебно-методические материалы (УМК / ПК)
- Поддерживается загрузка УМК (PDF, DOC/DOCX, PPT/PPTX) и ПК (PDF, JPG, PNG).
- Файлы сохраняются локально в `/uploads/*` и регистрируются в таблице `files` в PostgreSQL.
- API операции защищены Bearer-токенами и привязаны к пользователю (ID загрузчика).

### Тестирование (GIFT Формат)
Для гибкого создания тестов и анкет интегрирован собственный парсер **GIFT (General Import Format Technology)**.
- **Возможности:** парсинг вопросов со множественным выбором (`{~неверно =верно}`), True/False (`{T}` / `{F}`), кратких ответов и эссе (`{}`).
- **Использование:** Загрузка GIFT-тестов через `POST /files/gift/test/upload` и опросников экспертов через `POST /files/gift/form/upload` (модели `GiftTest` и `GiftForm`).
